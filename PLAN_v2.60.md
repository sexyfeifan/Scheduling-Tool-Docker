## 一、用户必须要求（硬性约束）

以下要求在整个迭代过程中**不可违背**：

| # | 要求 | 说明 |
|---|------|------|
| **R1** | 主视觉保持现状 | 当前周视图日历是**唯一主视图**，不做任何改动 |
| **R2** | 新视图均为辅助 | 月视图、人员排期、数据看板等通过**视图切换器**进入，用户可选 |
| **R3** | 视图可自由切换 | 导航栏提供视图切换入口，一键切换，一键返回主视图 |
| **R4** | 分支隔离 | 所有改动在 `v2.60-dev` 分支，不影响 GitHub `main` 正式版 |
| **R5** | 人员排期视图 | 独立页面，按人员×日期展示，支持筛选和冲突检测 |
| **R6** | iCal 日历订阅 | 排期数据 → 推送到外部日历（单向），提供订阅链接 |
| **R7** | 数据兼容 | 不修改现有数据模型字段，只新增可选字段 |
| **R8** | 测试通过 | 每个 Phase 完成后测试全部通过才能进入下一个 |

### 2.2 视图切换器 UI

在现有导航栏右侧添加一个视图切换器（不改动现有按钮布局）：

```
┌──────────────────────────────────────────────────────────────┐
│  ◀ 上一周 │ 2026年6月23日 - 6月29日 │ 下一周 ▶ │ 今天 │       │
│                                                              │
│  [+ 新增] [🖼 导出] [↩ 撤销] [🔍 搜索...] [🔥 热力图]        │
│                                                              │
│  ┌─────────────────────────────────────────┐ [📅周][📆月]    │
│  │                                         │ [👥人员][📊看板]│
│  │         周视图日历（不变）                │                 │
│  │                                         │                 │
│  └─────────────────────────────────────────┘                 │
└──────────────────────────────────────────────────────────────┘
```

- 默认选中「📅 周」
- 点击其他按钮切换到对应视图
- 点击「📅 周」回到主视图
- 切换时**不刷新页面**，纯前端切换
