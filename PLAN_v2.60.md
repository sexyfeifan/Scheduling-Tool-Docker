## 二、关于日历订阅（iCal）的澄清

### 方向：项目排期 → 推送到外部日历

```
┌──────────────┐    导出 .ics     ┌──────────────┐
│  排片系统     │ ──────────────→ │  手机/电脑日历 │
│  (数据源)     │                 │  (订阅目标)   │
└──────────────┘                 └──────────────┘
```

**具体行为：**
- 系统提供一个订阅链接：`/api/calendar.ics`
- 用户在手机日历（iOS/Android/Outlook/Google Calendar）中添加此订阅
- 日历中自动显示排期项目，每个项目 = 一个日历事件
- **数据流向**：排片系统 → 外部日历（单向推送）
- 日历自动刷新（可设 1 小时间隔）

**每个日历事件包含：**
```
标题：项目名称（类型）
时间：当天 startTime 起，预估 2 小时（或全天事件）
地点：拍摄场地
描述：导演 / 摄影师 / 制片 / 状态
颜色：按项目类型区分
```

**不是**从外部日历拉取数据到系统中。

## 三、新增功能：人员排期视图

### 3.1 功能定义

在主导航栏添加「👥 人员排期」按钮，点击进入独立视图：

```
            | 周一 6/23 | 周二 6/24 | 周三 6/25 | 周四 6/26 | 周五 6/27 |
────────────┼───────────┼───────────┼───────────┼───────────┼───────────┤
🎬 导演     |           |           |           |           |           |
  张导      | 项目A(视频)| 项目A(视频)|           | 项目D(平面)|           |
  李导      |           | 项目B(外拍)| 项目B(外拍)|           | 项目E(直播)|
────────────┼───────────┼───────────┼───────────┼───────────┼───────────┤
📷 摄影师   |           |           |           |           |           |
  王摄      | 项目A(视频)| 项目A(视频)| 项目C(试做)|           |           |
  赵摄      |           | 项目B(外拍)|           |           | 项目E(直播)|
────────────┼───────────┼───────────┼───────────┼───────────┼───────────┤
🎤 录音     |           |           |           |           |           |
  刘录      |           |           |           |           | 项目E(直播)|
```

### 3.2 核心能力

| 能力 | 说明 |
|------|------|
| **人员自动聚合** | 从所有项目中提取人员名单，按职能分组 |
| **日期范围选择** | 支持周视图 / 双周视图 / 月视图 |
| **筛选** | 按职能筛选（只看导演 / 只看摄影师） |
| **搜索** | 输入人名快速定位 |
| **冲突高亮** | 同一人同一天有 2+ 项目 → 红色标注 |
| **工作量统计** | 每人右侧显示本周项目数 |
| **点击跳转** | 点击单元格中的项目 → 跳转到主视图对应日期 |

### 3.3 数据来源

直接读取现有 `scheduleData`，无需新增后端接口。前端遍历所有日期的所有项目，按人员字段聚合。
