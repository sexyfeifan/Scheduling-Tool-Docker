#### 功能① 月视图甘特图

**解决的问题**：周视图只能看 7 天，跨周项目被截断，无法总览月度排期

**用户场景**：制片需要看某导演整个月的排期密度，或某场地整月的占用情况

**界面设计**：
```
┌────────────────────────────────────────────────────────────────┐
│  📅 2026年6月                         ◀ 6月 ▶   [返回周视图]    │
├─────┬─────┬─────┬─────┬─────┬─────┬─────┬─────┬─────┬─────────┤
│     │ 一  │ 二  │ 三  │ 四  │ 五  │ 六  │ 日  │     │         │
├─────┼─────┼─────┼─────┼─────┼─────┼─────┼─────┼─────┼─────────┤
│ 第1周│ ████████████ 项目A(视频)                              │
│     │           ██████ 项目B(外拍)                           │
├─────┼─────┼─────┼─────┼─────┼─────┼─────┼─────┼─────┼─────────┤
│ 第2周│     ██████████████ 项目C(试做)                        │
│     │ ████████ 项目D(平面)                                   │
├─────┼─────┼─────┼─────┼─────┼─────┼─────┼─────┼─────┼─────────┤
│ 第3周│                     ████████████████ 项目E(直播)       │
│     │                         ██████ 项目F(视频)             │
└─────┴─────┴─────┴─────┴─────┴─────┴─────┴─────┴─────┴─────────┘

- 每行 = 一个项目，横向条形跨越多个日期格
- 颜色按项目类型区分：视频蓝 / 外拍绿 / 试做橙 / 平面紫 / 直播红
- 鼠标悬浮显示项目详情（人名、场地、状态）
- 点击项目 → 切换到周视图并定位到该项目所在日期
```

## 第七部分：风险评估与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 人员排期视图性能问题 | 项目数量大时渲染卡顿 | 虚拟滚动：只渲染可视区域的行 |
| iCal 格式兼容性 | 部分日历客户端解析异常 | 严格遵循 RFC 5545 规范，重点测试 Apple/Google/Outlook |
| 月视图甘特图跨月项目 | 跨月项目的条形如何显示 | 按自然月截断，跨月项目在两个月都显示 |
| 视图切换内存占用 | 多个视图同时在 DOM 中 | 非激活视图销毁 DOM，切换时重新渲染 |
| 批量操作误删 | 用户批量删除后无法恢复 | 批量操作前弹确认框，保留撤销能力 |

## 第八部分：技术决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| 日历方案 | iCal (.ics) 订阅链接 | 所有日历客户端原生支持，无需 API 对接 |
| iCal 依赖库 | ical-generator（纯 JS，无原生依赖） | ARM64 兼容性好，无编译问题 |
| 月视图甘特图 | 纯 CSS 条形 + JS 渲染 | 不引入图表库，保持轻量 |
| 智能提醒依赖库 | node-cron（纯 JS） | 轻量级，ARM64 无兼容问题 |
| Excel 导出依赖库 | exceljs（纯 JS） | 不依赖原生模块，支持样式 |
| PWA 图标 | 使用 emoji 生成占位图标 | 后续可替换为正式设计稿 |
| 视图切换策略 | DOM 销毁/重建（非 CSS 隐藏） | 减少内存占用，避免隐藏视图中的事件泄漏 |
| 多角色视图实现 | URL 参数 + 前端过滤（非独立页面） | 代码复用率高，无需重复渲染逻辑 |

| 多角色视图实现 | URL 参数 + 前端过滤（非独立页面） | 代码复用率高，无需重复渲染逻辑 |

#### 功能⑤ 多角色视图

**解决的问题**：所有人看到相同的界面，不同角色的关注点不同

**用户场景**：导演只想看自己参与的项目，场地管理员只想看某个场地的占用情况

**设计思路**：不新建页面，在现有周视图上叠加过滤层，通过 URL 参数切换

```
默认：     /                    → 全部项目（当前行为，不变）
导演视角： /?view=director&name=张导  → 只显示张导参与的项目
场地视角： /?view=location&name=棚A  → 只显示棚A的项目
制片视角： /?view=producer           → 全部项目 + 状态统计叠加
```

## 第八部分：技术决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| 日历方案 | iCal (.ics) 订阅链接 | 所有日历客户端原生支持，无需 API 对接 |
| iCal 依赖库 | ical-generator（纯 JS，无原生依赖） | ARM64 兼容性好，无编译问题 |
| 月视图甘特图 | 纯 CSS 条形 + JS 渲染 | 不引入图表库，保持轻量 |
| 智能提醒依赖库 | node-cron（纯 JS） | 轻量级，ARM64 无兼容问题 |
| Excel 导出依赖库 | exceljs（纯 JS） | 不依赖原生模块，支持样式 |
| PWA 图标 | 使用 emoji 生成占位图标 | 后续可替换为正式设计稿 |
| 视图切换策略 | DOM 销毁/重建（非 CSS 隐藏） | 减少内存占用，避免隐藏视图中的事件泄漏 |
| 多角色视图实现 | URL 参数 + 前端过滤（非独立页面） | 代码复用率高，无需重复渲染逻辑 |

| 日历方案 | iCal (.ics) 订阅链接 | 所有日历客户端原生支持，无需 API 对接 |

| Excel 导出依赖库 | exceljs（纯 JS） | 不依赖原生模块，支持样式 |
| PWA 图标 | 使用 emoji 生成占位图标 | 后续可替换为正式设计稿 |
| 视图切换策略 | DOM 销毁/重建（非 CSS 隐藏） | 减少内存占用，避免隐藏视图中的事件泄漏 |
| 多角色视图实现 | URL 参数 + 前端过滤（非独立页面） | 代码复用率高，无需重复渲染逻辑 |

## 第八部分：技术决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| 日历方案 | iCal (.ics) 订阅链接 | 所有日历客户端原生支持，无需 API 对接 |
| iCal 依赖库 | ical-generator（纯 JS，无原生依赖） | ARM64 兼容性好，无编译问题 |
| 月视图甘特图 | 纯 CSS 条形 + JS 渲染 | 不引入图表库，保持轻量 |
| 智能提醒依赖库 | node-cron（纯 JS） | 轻量级，ARM64 无兼容问题 |
| Excel 导出依赖库 | exceljs（纯 JS） | 不依赖原生模块，支持样式 |
| PWA 图标 | 使用 emoji 生成占位图标 | 后续可替换为正式设计稿 |
| 视图切换策略 | DOM 销毁/重建（非 CSS 隐藏） | 减少内存占用，避免隐藏视图中的事件泄漏 |
| 多角色视图实现 | URL 参数 + 前端过滤（非独立页面） | 代码复用率高，无需重复渲染逻辑 |

| 决策 | 选择 | 理由 |
|------|------|------|
| 日历方案 | iCal (.ics) 订阅链接 | 所有日历客户端原生支持，无需 API 对接 |

| 智能提醒依赖库 | node-cron（纯 JS） | 轻量级，ARM64 无兼容问题 |
| Excel 导出依赖库 | exceljs（纯 JS） | 不依赖原生模块，支持样式 |
| PWA 图标 | 使用 emoji 生成占位图标 | 后续可替换为正式设计稿 |
| 视图切换策略 | DOM 销毁/重建（非 CSS 隐藏） | 减少内存占用，避免隐藏视图中的事件泄漏 |
| 多角色视图实现 | URL 参数 + 前端过滤（非独立页面） | 代码复用率高，无需重复渲染逻辑 |
